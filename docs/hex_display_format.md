# 十六进制数据显示格式说明

## 概述

TCP调试工具在显示二进制数据时，采用标准的十六进制编辑器格式，便于查看和分析数据的内存布局。

## 显示格式

```
0000  CC DD B0 01 00 00 0D BE 7C                        ........|
```

### 格式组成部分

| 部分 | 示例 | 说明 |
|------|------|------|
| **行偏移地址** | `0000` | 该行数据在整个数据包中的起始偏移位置（十六进制） |
| **十六进制数据** | `CC DD B0 01 00 00 0D BE 7C` | 实际的数据内容，以十六进制表示 |
| **ASCII字符** | `........\|` | 对应的ASCII字符显示，不可显示字符用`.`代替 |

## 详细说明

### 行偏移地址（4位十六进制）

- 表示该行第一个字节在整个数据中的位置
- 从 `0000` 开始计数
- 每行显示16个字节，所以第二行的偏移量是 `0010`（即十进制的16）

### 示例：长数据分行显示

```
0000  CC DD B0 01 00 00 0D BE 7C 11 22 33 44 55 66 77  ........|."3DUfw
0010  88 99 AA BB CC DD EE FF                          ........
```

- 第一行：偏移 `0000`（第0字节）到 `000F`（第15字节）
- 第二行：偏移 `0010`（第16字节）开始

### ASCII字符区域说明

| 字符范围 | 显示方式 |
|---------|---------|
| 可打印字符（0x20-0x7E） | 直接显示字符 |
| 控制字符、非ASCII | 显示为 `.` |

## 实际应用示例

### 发送数据示例
```
[16:37:06.279] [发送] [二进制数据]
0000  CC DD B0 01 00 00 0D BE 7C                        ........|
```

### 接收数据示例
```
[16:37:06.318] [二进制数据]
0000  AA BB B0 01 00 1E 0D DC B8                        ........
```

## 相关功能

- **十六进制发送**：勾选"十六进制发送"选项，可以发送十六进制字符串
- **十六进制显示**：勾选"十六进制显示"选项，接收数据以十六进制格式显示
- **数据转换**：工具自动处理文本和十六进制之间的转换

## 参考

- 十六进制（Hexadecimal）：基数为16的计数系统，使用数字0-9和字母A-F
- 偏移量（Offset）：数据相对于起始位置的字节数
