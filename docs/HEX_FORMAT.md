# TCP/UDP调试工具 - 十六进制格式说明

## 支持的十六进制输入格式

本工具支持多种十六进制数据输入格式，方便用户根据不同场景使用。

### 1. 标准格式（空格分隔）
```
56 39 39 47 30 30 30 38 0D
```
- 每个字节用空格分隔
- 最常用格式

### 2. 带h/H后缀格式
```
56h 39h 39h 47h 30h 30h 30h 38h 0Dh
```
或
```
56H 39H 39H 47H 30H 30H 30H 38H 0DH
```
- 每个字节后添加 `h` 或 `H` 后缀
- 支持大小写混合

### 3. 连续格式（无分隔）
```
56393947303030380D
```
- 所有字节连续输入
- 自动按每2个字符解析为一个字节

### 4. 0x前缀格式
```
0x56 0x39 0x39 0x47 0x30 0x30 0x30 0x38 0x0D
```
或
```
0X56 0X39 0X39 0X47 0X30 0X30 0X30 0X38 0X0D
```
- 每个字节前添加 `0x` 或 `0X` 前缀
- 支持大小写混合

## 使用示例

### 示例1：标准格式发送
```
勾选"十六进制发送"，输入：
48 65 6C 6C 6F 20 57 6F 72 6C 64
发送内容为："Hello World"
```

### 示例2：带h后缀格式发送
```
勾选"十六进制发送"，输入：
48h 65h 6Ch 6Ch 6Fh 20h 57h 6Fh 72h 6Ch 64h
发送内容为："Hello World"
```

### 示例3：混合使用
```
勾选"十六进制发送"，输入：
48h 65H 6C 6C 6F 0x20 57 6F 72 6C 64
发送内容为："Hello World"
```

## 注意事项

1. **格式优先级**：如果输入中包含 `h` 或 `H` 后缀，工具会优先按此格式解析
2. **大小写不敏感**：十六进制字符 `A-F` 和 `a-f` 都可以使用
3. **自动补零**：单字符字节（如 `5h`）会自动补零为 `05`
4. **无效字符**：会自动忽略空格、换行等非十六进制字符

## 接收数据显示

接收数据时，如果勾选"十六进制显示"，会显示为以下格式：
```
[14:30:25.123]
0000  48 65 6C 6C 6F 20 57 6F 72 6C 64                 Hello World
```
- 左侧为偏移地址
- 中间为十六进制数据
- 右侧为对应的ASCII字符（不可显示字符显示为`.`）
